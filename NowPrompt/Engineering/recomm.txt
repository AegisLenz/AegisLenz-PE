아래에 설명된 질문 주제를 참고하여 3개의 구체적이고 다양한 질문을 생성하세요. 각 질문은 해당 주제와 관련된 내용을 다루어야 하며, 주제에서 벗어난 질문은 포함하지 마세요.

### 주의사항
1. 생성된 질문은 사용자가 ChatGPT에게 다시 질문하여 공격을 심층적으로 분석할 수 있는 형태여야 합니다. **새로운 세부사항을 밝혀낼 수 있는 분석형 질문을 작성하세요**.
2. 질문 형식은 **구체적인 데이터를 요구하는 "~해주세요" 형태로 작성**하세요. 
3. 생성하는 질문은 **텍스트만 포함**하며, 큰따옴표("")를 사용하여 세 개의 질문을 구분하세요.
4. 생성된 질문은 구체적이고 다양한 정보를 요청해야 하며, "해당"이라는 단어 대신 구체적 필드명을 사용하세요.
5. 생성하는 질문은 반드시 하나의 주제만 포함하여 작성하세요. 질문에 절대 "하고"와 같은 연결어를 사용하지 마세요.
6. 생성하는 질문에는 "최소 권한을 만들어줘"와 같은 요청을 포함하지 마세요.

### 질문 주제 설명 :
1. "ES": <이는 AWS CloudTrail 로그에 관한 질문을 처리하는 것입니다. ElasticSearch에는 AWS CloudTrail 로그가 저장되어 있습니다.  
            따라서 ES는 질의를 분석하여 이에 알맞는 ElasticSearch 쿼리를 반환하는 기능을 합니다. 
            CloudTrail 로그는 AWS 계정 내에서 이루어지는 다양한 사용자 활동과 API 호출 기록을 수집하고 추적할 수 있도록 돕는 서비스입니다. 
            CloudTrail 로그를 통해 사용자의 행위를 분석하여 특정 액션의 발생 여부, 액세스된 리소스, 수행된 명령 등을 확인할 수 있습니다. 
            ElasticSearch 관련 질문은 검색 및 데이터 분석 작업을 다루며, ElasticSearch를 활용하여 CloudTrail 로그를 필터링하고 질의하거나, 특정 조건에 따라 데이터를 검색하는 방법에 중점을 둡니다. >
2. "DB": <AWS Cloud 자산에 대한 질문을 처리하는 것입니다. MongoDB에 AWS Cloud 자산이 저장되어 있으며, IAM 계정, EC2 인스턴스, S3 버킷과 같은 자산에 대한 상태와 속성정보 데이터를 관리합니다. 
            사용자는 다음과 같은 자산 정보에 대해 질의하고 이를 관리할 수 있습니다.
            IAM 사용자 데이터에는 사용자 이름(UserName), 사용자 ID(UserId), 생성일(CreateDate), 사용자 정책(UserPolicies), 연결된 정책(AttachedPolicies), 그룹(Groups), 비밀번호 마지막 사용 시점(PasswordLastUsed) 즉 마지막으로 해당 IAM 유저가 접속한 시간, 액세스 키가 사용된 기록인 이전 시간들(AccessKeysLastUsed), 마지막 업데이트 시점(LastUpdated)과 같은 정보가 포함되어 있습니다. 
            EC2 인스턴스 데이터는 인스턴스 ID(InstanceId), 인스턴스 유형(InstanceType), 인스턴스가 가장 처음 시작된 시간(LaunchTime), 상태(State), 공용 IP 주소(PublicIpAddress), 사설 IP 주소(PrivateIpAddress), VPC ID(VpcId), 서브넷 ID(SubnetId), 보안 그룹(SecurityGroups), 태그(Tags), EBS 볼륨(EbsVolumes), 네트워크 인터페이스(NetworkInterfaces), IAM 인스턴스 프로파일(IamInstanceProfile) 등의 필드를 포함합니다. 예를 들어, "현재 'stopping' 상태의 EC2 인스턴스 목록을 보여주세요"와 같은 질의가 해당됩니다.
            S3 버킷 데이터는 버킷 이름(Name), 생성일(CreationDate), 위치(Location), 액세스 제어 목록(ACL), 정책(Policy), 로깅 설정(Logging), 버전 관리 상태(Versioning), 태그(Tags) 등의 정보를 다룹니다.
            또한 사용자의 마지막 접속 시간 등이 존재합니다. 
            예를 들어, "현재 'stopping' 상태의 EC2 인스턴스 목록을 보여주세요"와 같은 질의가 해당됩니다. >
3. "Policy": <IAM 계정 권한과 관련하여 최소권한과 현재 권한과의 비교에 대한 질문을 처리합니다. 
                "Policy"가 다양한 사용자의 권한에 접근하는 것이 아닙니다. 
                이 서비스는 권한 사이 비교 및 권한 자체에 대한 질문을 처리해야합니다. 
                현재 서비스는 90일 동안의 CloudTrail 기반으로 IAM 계정이 사용한 eventName을 바탕으로 이를 발생시킬 수 있는 권한을 추출하여 최소 권한 정책을 생성하고, 해당 IAM 계정의 현재 권한과 비교해주는 기능을 제공하고 있습니다. 
                "Policy" 주제는 이러한 권한 비교 설명과 함께 AWS IAM 계정 권한 정책과 관련된 질문을 다룹니다. 
                정책 비교와 관련된 설명이나 AWS IAM 정책 관리에 대한 질문도 이 주제에 포함됩니다. 
                이를 통해 보다 안전하고 최적화된 IAM 정책 관리에 대한 답변을 제공합니다. >
